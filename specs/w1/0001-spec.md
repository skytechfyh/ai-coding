# Project Alpha 需求和设计文档

## 1. 项目概述

### 1.1 项目背景

Project Alpha 是一个基于标签分类的 Ticket 管理系统，旨在帮助用户高效地管理和组织各类任务或问题单。该系统采用前后端分离架构，后端使用 Fast API 框架，前端使用 TypeScript/Vite/Tailwind CSS/Shadcn UI 构建。

### 1.2 项目目标

- 提供简洁高效的 Ticket 创建、编辑、删除操作
- 支持通过标签对 Ticket 进行分类管理
- 提供灵活的 Ticket 筛选和搜索功能
- 构建用户友好的 Web 界面
- 无需用户系统，单用户直接使用

### 1.3 技术栈

| 层级 | 技术选型 | 版本 |
|------|----------|------|
| 数据库 | PostgreSQL | 15+ |
| 后端框架 | Fast API | 0.109+ |
| Python | Python | 3.11+ |
| ORM | SQLAlchemy | 2.0+ |
| 数据验证 | Pydantic | 2.5+ |
| 前端框架 | React | 18+ |
| 构建工具 | Vite | 5+ |
| UI 框架 | Tailwind CSS | 3.4+ |
| 组件库 | Shadcn UI | 最新 |
| 状态管理 | TanStack Query | 5+ |
| HTTP 客户端 | Axios | 1.6+ |
| 表单处理 | React Hook Form | 7+ |
| 表单验证 | Zod | 3+ |

### 2.2 用户交互流程

#### 2.2.1 创建 Ticket 流程

```
用户点击"新建 Ticket"按钮
    ↓
弹出创建表单对话框
    ↓
用户填写标题（必填）、描述（可选）、选择标签（可选）
    ↓
用户点击"创建"按钮
    ↓
系统验证表单数据（标题不能为空，最大255字符）
    ↓
创建成功 → 关闭对话框 → 刷新列表 → 显示成功提示 (Toast)
创建失败 → 显示错误提示
```

#### 2.2.2 编辑 Ticket 流程

```
用户在 Ticket 卡片上点击"编辑"按钮
    ↓
弹出编辑表单对话框（预填现有数据）
    ↓
用户修改标题、描述、标签
    ↓
用户点击"保存"按钮
    ↓
系统验证并更新数据
    ↓
保存成功 → 关闭对话框 → 刷新列表 → 显示成功提示
保存失败 → 显示错误提示
```

#### 2.2.3 删除 Ticket 流程

```
用户在 Ticket 卡片上点击"删除"按钮
    ↓
弹出确认对话框（"确定要删除这个 Ticket 吗？"）
    ↓
用户点击"确认删除"
    ↓
系统删除数据
    ↓
删除成功 → 刷新列表 → 显示成功提示
删除失败 → 显示错误提示
```

#### 2.2.4 标签筛选流程

```
用户在侧边栏点击某个标签
    ↓
系统筛选出包含该标签的所有 Ticket
    ↓
列表视图更新为筛选结果
    ↓
高亮当前选中的标签
    ↓
用户可点击"全部"或再次点击标签清除筛选
```

#### 2.2.5 搜索流程

```
用户在搜索框输入关键字
    ↓
用户按下回车键或点击搜索按钮（可选实时搜索）
    ↓
系统发送搜索请求
    ↓
显示搜索结果
    ↓
用户可清空搜索框恢复显示全部
```

### 2.3 数据验证规则

#### Ticket 验证规则

| 字段 | 规则 | 错误提示 |
|------|------|----------|
| title | 必填，1-255 字符 | 标题不能为空，标题不能超过 255 字符 |
| description | 可选，最大 5000 字符 | 描述不能超过 5000 字符 |
| status | 枚举值：pending, completed | 状态值无效 |
| tag_ids | 可选，数组，每个值必须是存在的标签 ID | 标签不存在 |

#### Tag 验证规则

| 字段 | 规则 | 错误提示 |
|------|------|----------|
| name | 必填，1-50 字符，唯一 | 标签名称不能为空，标签名称不能超过 50 字符，标签名称已存在 |
| color | 可选，7 字符十六进制颜色值，默认 #6B7280 | 颜色格式无效 |

### 2.4 边界情况处理

| 场景 | 处理方式 |
|------|----------|
| 删除标签时该标签已被 Ticket 使用 | 允许删除，自动解除关联关系 |
| 创建重复名称的标签 | 返回 409 错误，提示名称已存在 |
| 请求不存在的 Ticket | 返回 404 错误 |
| 请求不存在的标签 | 返回 404 错误 |
| 数据库连接失败 | 返回 500 错误，显示友好的错误信息 |
| 网络请求超时 | 显示超时提示，允许重试 |

```python
def get_db():
```

#### 4.4.2 Ticket 模型 (models/ticket.py)

```python
# backend/app/models/ticket.py
from sqlalchemy import Column, Integer, String, Text, DateTime, ForeignKey, Table
from sqlalchemy.orm import relationship
from datetime import datetime
from app.database import Base

ticket_tags = Table(
    'ticket_tags',
    Base.metadata,
    Column('ticket_id', Integer, ForeignKey('tickets.id', ondelete='CASCADE'), primary_key=True),
    Column('tag_id', Integer, ForeignKey('tags.id', ondelete='CASCADE'), primary_key=True)
)

class Ticket(Base):
    __tablename__ = 'tickets'

    id = Column(Integer, primary_key=True, index=True)
    title = Column(String(255), nullable=False)
    description = Column(Text, nullable=True)
    status = Column(String(20), default='pending', nullable=False)
    created_at = Column(DateTime, default=datetime.utcnow, nullable=False)
    updated_at = Column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow, nullable=False)

    tags = relationship('Tag', secondary=ticket_tags, back_populates='tickets')
```

#### 4.4.3 Tag 模型 (models/tag.py)

```python
# backend/app/models/tag.py
from sqlalchemy import Column, Integer, String, DateTime
from sqlalchemy.orm import relationship
from datetime import datetime
from app.database import Base

class Tag(Base):
    __tablename__ = 'tags'

    id = Column(Integer, primary_key=True, index=True)
    name = Column(String(50), unique=True, nullable=False, index=True)
    color = Column(String(7), default='#6B7280', nullable=False)
    created_at = Column(DateTime, default=datetime.utcnow, nullable=False)

    tickets = relationship('Ticket', secondary='ticket_tags', back_populates='tags')
```
