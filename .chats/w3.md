# Instructions

## code review command
帮我参照 @.claude/commands/speckit.specify.md 的结构，think ultra hard 构建一个对Python 和 TypeScript 的代码进行深度审查的命令，放在@.claude/commands/下，主要考虑以下几个方面：

- 架构和设计: 是否考虑 python 和 typescript 的架构和设计的最佳实践？是否有清晰的接口设计？是否考虑一定程度的可扩展性和可维护性？
- KISS 原则: 是否遵循 KISS 原则？代码是否过于复杂？是否有冗余的代码？是否有过度设计的部分？
- 代码质量: DRY, YAGNI, SOLID , etc. 是否有清晰的命名？是否有适当的注释？是否有一致的代码风格？是否有潜在的错误或漏洞？
- 性能: 是否有性能问题？是否有不必要的计算或资源消耗
- builder 模式: 是否有使用 builder 模式来构建复杂对象的部分？是否有过于复杂的构建过程？是否有不必要的步骤？
